# **B2B 도매 위탁판매 플랫폼의 API 데이터 연동 체계 및 상품 수집 대체 방법론에 관한 심층 연구 보고서**

## **1\. 서론: 전자상거래 환경에서의 B2B 도매 위탁판매와 데이터 연동의 전략적 중요성**

현대의 디지털 전자상거래 생태계, 특히 재고를 직접 보유하지 않고 공급사의 인프라를 활용하는 무재고 위탁판매(Dropshipping) 비즈니스 모델에서는 다수의 상품 데이터를 얼마나 신속하고 정확하게 소싱하여 개별 판매 채널에 연동하는지가 기업의 핵심 경쟁력으로 작용한다. 과거의 전통적인 전자상거래 환경에서 위탁판매자는 도매 사이트에서 개별 상품의 이미지와 상세페이지, 가격 정보를 수동으로 다운로드하여 스마트스토어나 오픈마켓에 재업로드하는 노동 집약적인 방식을 취했다. 그러나 취급하는 상품 데이터베이스(DB) 단위, 즉 SKU(Stock Keeping Unit)가 수천에서 수만 개로 기하급수적으로 증가함에 따라, 수동 관리 아키텍처의 한계가 명확하게 드러나게 되었다. 상품의 일시 품절, 단가 변동, 옵션 체계의 변경 등의 실시간 업데이트 지연은 곧 판매자의 페널티 누적 및 소비자 불만으로 직결되는 치명적인 운영 리스크를 유발한다.

이러한 운영상의 병목 현상과 데이터 불일치 문제를 해결하기 위해 등장한 기술적 대안이 바로 응용 프로그램 프로그래밍 인터페이스(Application Programming Interface, 이하 API)를 통한 이기종 시스템 간의 직접 연동이다. 국내 주요 B2B 도매 플랫폼들은 자사의 방대한 상품 데이터베이스를 외부 판매자들의 독립 쇼핑몰이나 서드파티 통합 관리 솔루션과 직접 연결할 수 있도록 OpenAPI를 제공하거나, 자체적인 전용 데이터 송출 클라이언트 프로그램을 무상으로 배포하는 생태계를 구축하고 있다. 이를 통해 상품의 대량 등록, 재고 동기화, 발주 처리, 운송장 번호 입력에 이르는 전자상거래의 전 주기적(Full-cycle) 업무가 극도로 자동화되는 추세이다.

본 보고서는 국내 B2B 도매 위탁판매 시장을 주도하는 주요 플랫폼들의 생태계 리스트를 카테고리별로 점검하고, 각 플랫폼이 제공하는 API의 기술적 사양, 지원 기능, 그리고 세부적인 과금 체계(유료 및 무료 여부)를 심층적으로 분석한다. 더불어 플랫폼 자체에서 API를 지원하지 않거나 판매자의 기술적 한계로 API를 직접적으로 활용하기 어려운 경우, 이를 대체하여 상품 데이터를 대량으로 수집하고 관리할 수 있는 다층적인 방법론 및 서드파티 솔루션(SaaS) 생태계까지 포괄적으로 조망하여, 판매자 및 유통 전문가들에게 전략적 통찰을 제공하고자 한다. 궁극적으로 본 연구의 결과는 사용자의 요구에 부합하도록 최종 종합 엑셀표 형식의 마크다운 테이블로 치밀하게 구조화되어 제시될 것이다.

## **2\. 국내 주요 B2B 도매 위탁판매 사이트 생태계 및 최신 순위 분석**

국내 B2B 도매 시장은 고도의 인프라를 갖춘 소수의 대형 종합 플랫폼과 특정 카테고리에 특화되어 깊이 있는 상품군을 제공하는 버티컬(Vertical) 플랫폼으로 양분되어 진화하고 있다. 국내 도매사이트 순위 및 정보를 전문적으로 집계하는 셀러위키(Seller-Wiki)의 주간 B2B 랭킹 데이터 트렌드 분석에 따르면, 현재 시장에는 약 300여 곳 이상의 도매 사이트가 활발히 운영되고 있다. 이들 플랫폼은 취급하는 상품의 스펙트럼, 물류 및 배송 정책, 그리고 판매자를 위한 기술적 지원 수준인 API 개방성 및 엑셀 대량등록 지원 여부에서 뚜렷한 차별성을 보인다.

### **2.1. 대형 종합 B2B 플랫폼의 과점적 인프라 지배**

시장을 선도하며 데이터 연동의 표준을 제시하는 대표적인 종합 도매 플랫폼으로는 도매꾹/도매매, 오너클랜, 온채널, 도매토피아 등이 있다. 이들은 각각 수백만 개 단위의 상품 DB를 엑셀 형태 및 API 형태로 보유하고 있으며, 대규모 트래픽을 감당할 수 있는 안정적인 서버 인프라와 고도화된 연동 시스템을 갖추고 있어 초보 판매자부터 대형 벤더에 이르기까지 폭넓게 활용된다.

첫째, 도매꾹 및 도매매는 B2B 위탁판매의 가장 대표적인 중추 플랫폼으로, 판매자와 공급자 간의 거래를 중개할 뿐만 아니라 오픈 API를 가장 체계적이고 개방적으로 제공하여 국내 다수의 쇼핑몰 통합관리 솔루션들의 데이터 원천(Data Source) 역할을 수행하고 있다.

둘째, 온채널은 자체적인 상품 데이터센터를 구축하고 쇼핑몰별 API 연동을 고도화하여, 판매자의 수동 개입을 최소화하는 자동 발주 및 송장 연동 시스템을 강력하게 지원하며 생태계를 확장하고 있다.

셋째, 오너클랜은 방대한 상품군을 바탕으로 판매자 친화적인 자체 무료 대량등록 프로그램인 '다팔자'를 운영하며 강력한 생태계를 구축하고 있으며, 다수의 외부 서드파티 툴과의 연동을 허용하여 시장 점유율을 유지하고 있다.

넷째, 도매토피아는 수입 및 국내 상품, 산지 직송, 자체 제조 등 상품 코드(GT, GK, XT, GKM, GDR, GDF, GKD, ATS, AKS)를 매우 세분화하여 배송 기간과 출고처를 시스템적으로 관리하며, 대행사를 통한 API 연동 및 엑셀 대량 등록을 지원하는 고유의 물류 중심 생태계를 형성하고 있다.

### **2.2. 카테고리 특화 버티컬 플랫폼의 부상 및 시장 세분화**

대형 종합몰의 틈새를 공략하여 특정 품목의 전문성과 상대적으로 높은 마진율을 내세운 버티컬 도매 플랫폼들의 약진 또한 이커머스 트렌드의 중요한 축이다. 셀러위키의 B2B 카테고리 데이터에 따르면, 생활/건강, 패션의류 및 잡화, 화장품/미용, 디지털/가전, 가구/인테리어, 출산/육아, 식품, 반려동물 용품 등 고도로 세분화된 영역에서 두각을 나타내는 플랫폼들이 다수 존재한다.

이러한 버티컬 플랫폼들은 카테고리의 특성상 제조사와의 직거래를 통한 단가 경쟁력 확보에 유리하지만, IT 인프라 투자의 한계로 인해 대형 플랫폼 수준의 오픈 API 구축 비율이 상대적으로 낮다는 한계를 지닌다. 따라서 이들 플랫폼을 활용하는 판매자들은 상품 DB를 엑셀 형식으로 다운로드하여 수동 가공하거나, 웹 스크래핑(Web Scraping) 기반의 서드파티 솔루션에 의존하는 반자동 수집 방식을 채택하는 경향이 짙다.

## **3\. 핵심 B2B 플랫폼별 API 제공 현황, 기술 사양 및 상세 요금 체계 분석**

API는 플랫폼 비즈니스에서 시스템의 심장과도 같은 역할을 수행한다. 수시로 변동되는 공급사의 상품 데이터(재고, 단가, 옵션명)가 실시간으로 갱신되는 상황에서, 이를 판매자의 쇼핑몰로 지연 없이 전달하는 파이프라인이기 때문이다. 본 장에서는 데이터를 제공하는 주요 플랫폼들의 API 아키텍처 구조, 지원하는 연동 기능의 범위, 그리고 판매자의 재무적 의사결정에 가장 중요한 영향을 미치는 API 이용 요금 체계를 상세히 해부한다.

### **3.1. 도매매 및 도매꾹 (Domeggook / Domemae) OpenAPI 생태계**

도매꾹과 도매매는 하나의 아이디로 양쪽 플랫폼을 모두 이용할 수 있는 통합 계정 시스템을 채택하고 있으며, 외부 개발자 및 대량 판매자들의 비즈니스 확장성을 지원하기 위해 '도매꾹 OpenAPI'를 전면적으로 제공하고 있다. 이 시스템은 RESTful 아키텍처와 유사한 형태의 HTTP 요청 기반 통신을 지원하여 다양한 개발 언어에서 접근이 용이하다.

첫째, API 발급 및 기술적 접근 권한에 있어 엄격한 자격 증명(Authentication) 절차를 요구한다. 도매꾹 API를 사용하기 위해서는 최소 1개의 계정이 필수적이며, 상품 정보 조회 등 기본적인 기능은 개방되어 있으나, 상품을 대량으로 플랫폼에 전송하는 '상품 등록(setItemBatch API)'과 같은 핵심적인 트랜잭션 발생 API는 단순 개인 회원이 아닌 '사업자 인증'을 완벽히 마친 사업자 회원만이 호출할 수 있도록 제한을 두고 있다. 사용자는 로그인 후 관리 페이지 내 'API 키 관리' 메뉴에서 고유한 API Key를 직접 발급받아야 하며, 시스템 부하 분산 및 다중 채널 관리를 위해 계정 1개당 최대 5개의 독립적인 Key 생성을 허용한다.

둘째, 가장 중요한 비용 및 과금 구조 측면에서 도매꾹/도매매가 제공하는 OpenAPI 자체의 발급과 호출 권한에 대해서는 **명시적인 과금이나 월 정액제 가입 비용이 전혀 없는 전면 무료 정책**을 시행하고 있다. 나아가, 자체적인 개발 지식이 전무하여 API 코드를 직접 다룰 수 없는 일반 셀러들을 위해 도매매는 '스피드고전송기'라는 자체 개발 대량등록 프로그램을 별도의 비용 없이 무료로 배포하고 있다. 사용자는 도매매 페이지에서 발급받은 API 키를 복사하여 샤플링과 같은 외부 제휴 통합 솔루션의 고객사 정보 항목에 입력하거나, 스피드고전송기 내부의 스마트스토어 전송 템플릿 관리에 연동용 판매자 아이디를 등록함으로써 클릭 몇 번만으로 상품을 일괄 등록할 수 있는 환경을 누릴 수 있다.

셋째, 무료 제공에 따른 시스템 남용을 방지하기 위한 기술적, 운영적 통제 기준이 매우 구체적으로 명시되어 있다. 과도한 API 호출로 서버에 부하를 주거나, 비정상적인 탐색 활동, 가상사설망(VPN) 등을 활용하여 IP 주소를 은닉하는 보안 우회 행위가 적발될 경우 즉각적인 API 이용 제한 처리를 시행한다.

### **3.2. 온채널 (Onchannel) 상품 데이터센터 및 부가기능 과금 기반 API**

온채널은 도매매와 달리 단순한 상품 데이터 조회를 넘어, 스마트스토어, 쿠팡, 오픈마켓으로의 상품 등록부터 주문 수집, 포인트 자동 결제를 통한 발주, 그리고 송장 처리로 이어지는 무재고 위탁판매의 전 프로세스를 원스톱으로 자동화하는 데 초점을 맞춘 고도화된 API 시스템을 제공한다.

첫째, 온채널 API 연동 워크플로우는 4단계로 구성된다.

1단계로 판매자의 오픈마켓 API 키를 온채널에 등록하고, 2단계로 온채널 상품을 쇼핑몰에 바로 전송한다. 3단계는 공급사가 단가/품절을 변경하면 이를 마켓에 실시간 동기화하는 기능이다. 4단계 'API 주문 수집 및 자동 발주' 기능은 매일 새벽(오전 01시\~07시) 신규 주문 내역을 자동 수집하고, 오전 08시에서 09시 사이에 판매자가 충전해 둔 '온채널 포인트' 잔액을 차감하여 자동으로 발주를 넣고 송장까지 연동한다.

둘째, 온채널의 요금 체계는 '기본 기능 무료, 지적 부가 기능 유료화' 구조이다.

마켓으로의 단순 API 상품 전송, 주문 수집, 자동 발주 등 인프라 연동 기능 자체에 부과되는 월간 가입비나 수수료는 없으며 전면 무료로 이용 가능하다.

다만, 판매 전환율을 극대화하기 위한 전문 키워드 분석 솔루션 '판다키워드' 제휴 서비스는 별도의 유료 구독 모델이다. 상품 전송 시 마켓의 검색 경쟁 강도를 분석하여 최적화된 상품명을 자동으로 추천 및 생성해 주는 이 프리미엄 데이터 서비스의 이용료는 **월 4,900원**이다. 최초 가입 시 **첫 15일간 무료 체험** 혜택을 제공하며, 기간 만료 전 해지하지 않으면 충전된 포인트에서 매월 정기 결제된다.

### **3.3. 오너클랜 (Ownerclan)의 자체 솔루션 중심 생태계와 서드파티 간접 연동**

오너클랜은 수백만 개의 상품 데이터베이스를 외부 개방형 OpenAPI로 직접 풀기보다는, 자체 개발한 클라이언트 프로그램과 시장의 검증된 전문 서드파티 솔루션(SaaS)을 통한 간접 연동 생태계에 집중한다.

첫째, 오너클랜은 자체 개발한 초고속 대량등록 및 재고 관리 프로그램인 \*\*'다팔자'\*\*를 완전 **무료**로 배포하고 있다. 다팔자는 스마트스토어 및 쿠팡 연동에 특화되어 있으며, 방대한 상품 그룹을 빠르게 마켓에 등록하고 품절/단종 상품을 자동으로 판매 중지 처리하는 강력한 동기화 기능을 자랑한다. 또한, 하나의 오너클랜 계정으로 여러 마켓 아이디 연동을 지원하여 복수 사업자 관리에 유리하다.

둘째, 윈들리(Windly), 플레이오토, 사방넷 등과 같은 상용 서드파티 통합 쇼핑몰 솔루션을 활용하여 오너클랜 데이터를 연동할 수 있다. 윈들리와 같은 최신 솔루션은 윈들리 시스템 환경설정에서 오너클랜 일반 아이디/비밀번호를 입력하여 세션 인증을 통해 데이터를 수집한다. 단, 간편 로그인(카카오, 네이버) 계정은 연동이 원천 차단되므로 일반 계정으로 재가입해야 한다. 이러한 외부 솔루션 이용 시 오너클랜에 내는 비용은 없으나, 솔루션 개발사에 지불하는 라이선스 비용(**월 7만 원 \~ 30만 원 수준**)이 발생한다.

### **3.4. 도매토피아 (Dometopia) 시스템의 물류 코드 기반 데이터 전송 체계**

도매토피아는 인천 소재의 자체 거대 물류센터 인프라를 바탕으로 회원 등급제에 따른 적립금 지급 및 할인 혜택을 제공하는 생태계를 구축하고 있다. 상품 데이터는 당일 발송(GT/GK), 산지 직송(GKM), 해외 직구(GDR) 등 출고처와 배송 기간을 의미하는 상품 코드로 분류되어 전송된다.

API 연동에 있어서는 시스템 내부에서 고유 API ID를 생성한 후, 도매매와 연계된 제휴 시스템(주 ENG 커머스 등)에 판매자 ID를 맵핑하여 연결을 수행한다. 마진 요율과 배송비 설정을 템플릿화하여 상품 전송 시 일괄 적용할 수 있으며, API 키 발급 및 데이터 유지에 부과되는 별도의 월 정액 유지비는 없어 **기본 무료**로 제공된다.

**4\. API 미제공 및 인프라 연동 한계 시의 상품 수집 대체 기술 및 방법론**

수많은 중소형 B2B 도매 사이트나 특화 버티컬몰은 독자적인 OpenAPI 서버를 유지할 자본과 기술력이 부족하다. 이 경우 적용할 수 있는 대체 수집 방법론은 크게 3가지이다.

### **4.1. 플랫폼 제공 자체 전용 클라이언트 솔루션 무상 활용**

가장 대표적으로 오너클랜의 '다팔자', 도매매의 '스피드고전송기'가 있다. 100% 무료이며 해당 몰의 상품 체계에 완벽히 최적화되어 오류가 적다. 하지만 각 프로그램이 자사 몰의 데이터만 수집하므로 취급 도매처가 늘어날수록 관리가 파편화되는 단점이 있다.

### **4.2. 서드파티 구독형 통합 쇼핑몰 관리 솔루션(SaaS) 도입**

비즈니스의 스케일업을 위해 가장 필수적인 대안이다. 사방넷, 플레이오토, 스윕(Sweep), 윈들리(Windly), 셀파이, 케이셀러, 셀링콕 등 거대 데이터 허브 역할을 하는 솔루션들을 도입하는 것이다. 이들은 크롤링 봇(Bot)이나 제휴 통신망을 통해 수십 개의 B2B 사이트 데이터를 강제로 긁어와 여러 오픈마켓에 동시 송출한다.

진입 장벽은 **월 7만 원에서 최대 30\~50만 원에 이르는 유료 구독 모델**이다. 취급 SKU 용량과 연동 마켓 개수에 따라 계단식으로 요금이 증가한다.

### **4.3. 엑셀(Excel) 기반 데이터 대량 처리 및 일괄 주문 아키텍처**

솔루션 구독료를 아끼거나 폐쇄형 영세 B2B 사이트를 공략할 때 쓰이는 가장 원초적인 방법이다. 게시판에서 상품 DB 엑셀(.csv)을 다운로드하고, VLOOKUP이나 매크로 함수로 도매 원가에 마진율을 곱하여 재가공한 뒤, 오픈마켓 셀러 오피스에 일괄 업로드한다. 또한 도매꾹 등의 사이트에 주문자의 배송지 엑셀을 밀어 넣고 '이머니' 포인트 잔액으로 일괄 결제하는 수동 발주 자동화에도 쓰인다. 그러나 데이터 불일치(품절 업데이트 지연) 위험이 커 철저한 수동 재고 관리가 필수적이다.

**5\. 데이터 연동 및 상품 수집 자동화 적용 시 핵심 고려사항 및 리스크 매니지먼트 전략**

* **물류비 인상 및 규격 외 화물 역마진 리스크:** 도매처(예: 젠트레이드)에서 일방적으로 공지하는 이형 화물(160cm 초과 시 1만 원 추가 등) 할증 요금은 API 봇이 텍스트를 인지하지 못한다. 마켓 배송비를 일괄 3,000원으로 하드코딩해 두었다가 대형 화물이 팔리면 만 원 이상의 적자가 발생하므로 화물 코드는 전송에서 제외하는 세팅이 필요하다.  
* **지식재산권 침해 딜레마:** 기계적 자동 전송 시, 섞여 있는 유명 브랜드 가품이나 도용 이미지를 판매자가 인지하지 못해 내용증명을 받을 위험이 높다. 솔루션 내부의 '금칙어 사전' 필터링 기능을 반드시 활성화하여 상표권 키워드를 방어해야 한다.  
* **시스템 이중 트랜잭션 충돌 오류:** 온채널과 같이 매일 아침 자동으로 포인트를 차감하여 발주하는 시스템을 켜둔 상태에서, 전날 밤 판매자가 마음이 급해 수동으로 먼저 발주해 버리면, 기계는 이를 인지하지 못하고 아침에 또 발주를 넣어 고객에게 2개의 물건이 배송되는 중복 오류가 발생할 수 있다.

**6\. 결론 및 성공적인 도매 위탁 비즈니스를 위한 전략적 통찰**

국내 B2B 위탁판매 생태계는 데이터 개방성을 무기로 한 'API 초자동화' 영역으로 진입했다. 도매꾹, 온채널, 오너클랜은 파이프라인 이용 자체는 무료로 개방하여 셀러를 락인(Lock-in)시키고, 부가적인 분석 데이터(키워드 툴 등)에 과금하는 고도화된 전략을 쓴다. 초기 셀러는 100% 무료인 '다팔자'나 '스피드고전송기'로 마켓 감각을 익히고, 성장 궤도에 오르면 여러 롱테일 버티컬 사이트의 상품을 통합 스크래핑할 수 있는 유료 SaaS(사방넷, 플레이오토 등)에 투자하여 레버리지를 극대화해야 한다. 궁극적인 성공은 API라는 기술 그 자체가 아니라, 쏟아지는 데이터 속에서 지재권 리스크와 역마진을 필터링해 내는 판매자의 꼼꼼한 프로세스 통제 역량에 달려 있다.
