# 설계서와 실제 화면/시트 맞추기 (배포·동기화)

로컬에는 **시중가 조회, 소싱목록, 판매관리, 회계** 등이 구현된 새 버전이 있지만,
**Netlify에 올라간 파일**과 **구글 시트에 붙어 있는 Apps Script**가 예전 버전이면 화면과 시트가 설계서와 다르게 보입니다.

아래 순서대로 하면 **실제 웹·시트가 설계서(SYSTEM_DESIGN.md)와 같아집니다.**

---

## 1. 웹 페이지를 새 버전으로 배포 (Netlify)

**증상:** 계산 탭에 "시중가 조회" 버튼이 없고, 탭이 "상품목록·통계·설정"만 보임 (판매관리·회계 없음).

**이유:** Netlify에는 예전에 올렸던 `seller-dashboard-v3.html`만 배포된 상태입니다.

**조치:**

1. **로컬**에서 최신 `seller-dashboard-v3.html` 파일을 엽니다.
   (경로: 프로젝트 루트의 `seller-dashboard-v3.html`)
2. **Netlify** (https://app.netlify.com) 로그인 → 해당 사이트 선택.
3. **Sites → Deploys** (또는 **Drag and drop**) 에서
   **`seller-dashboard-v3.html` 파일을 드래그해서 배포**합니다.
   - "Deploy manually" 로 이 한 파일만 올려도 됩니다.
   - 기존에 "publish directory" 로 폴더를 올렸다면, 그 폴더 안의 `seller-dashboard-v3.html` 을 **이 로컬 파일로 교체**한 뒤 다시 배포하면 됩니다.
4. 배포가 끝나면 **같은 URL**로 접속해 보세요.
   - **계산** 탭에 "시중가 조회" 버튼
   - 탭에 **소싱목록, 판매관리, 회계**
   이 보이면 웹은 설계서와 맞춰진 것입니다.

---

## 2. 구글 시트 Apps Script를 확장 버전으로 교체

**증상:** 시트에 "카테고리, 경쟁강도, 시중최저가, 시중평균가, 판매결정, 판매시작일" 컬럼이 없고, "판매기록·매입매출" 시트도 없음.

**이유:** 시트에 붙어 있는 스크립트가 **13컬럼만 쓰는 예전 버전**입니다.
설정 탭의 "Apps Script 코드 복사"도 예전에는 이 예전 스크립트만 복사했습니다.

**조치:**

1. **구글 시트** 열기 (시트 ID: `1D6IlJquibWJfUkmIrKSz-PF4JYSa10dJd_GQdwtSSSg`).
2. **확장 프로그램 → Apps Script** 로 이동.
3. 기존 코드 **전부 삭제**.
4. **프로젝트 폴더**의 `apps-script/Code.gs` 파일을 열어서 **내용 전체를 복사**한 뒤, Apps Script 편집기에 **붙여넣기**.
5. **저장** (Ctrl+S).
6. **배포 → 새 배포 → 유형: 웹 앱**
   - 실행 사용자: **본인**
   - 액세스: **모든 사용자**
   → 배포 후 나오는 **웹 앱 URL**을 복사합니다.
7. **대시보드**에서 **설정** 탭 → "Step 3 — Apps Script 배포 URL"에 **방금 복사한 URL** 입력 → 저장.
8. **상품목록 시트 14~19열 헤더가 없으면** 다음 중 하나로 맞춥니다.
   - **방법 A (권장)**
     Apps Script 편집기에서 함수 목록에 **`setProductListHeaders`** 를 선택한 뒤 **실행** 버튼을 한 번 누릅니다.
     → N~S열 1행에 `카테고리`, `경쟁강도`, `시중최저가`, `시중평균가`, `판매결정`, `판매시작일` 이 들어갑니다.
   - **방법 B (수동)**
     1행에 아래처럼 입력합니다.
     - N1: `카테고리`
     - O1: `경쟁강도`
     - P1: `시중최저가`
     - Q1: `시중평균가`
     - R1: `판매결정`
     - S1: `판매시작일`
     R1에 `$` 등 다른 문자가 있으면 지우고 `판매결정`으로 바꿉니다.

이후 **상품 저장**을 한 번 하면 "판매기록" / "매입매출" 시트는 새 스크립트가 **자동으로 생성**합니다.

## 3. 백엔드 API URL (시중가 조회용)

**증상:** "시중가 조회" 버튼을 눌러도 동작하지 않거나 "API URL을 입력하세요" 메시지가 뜸.

**조치:**

1. 대시보드 **설정** 탭으로 이동.
2. **"백엔드 API URL (시중가 조회용)"** 란에
   Render에 배포한 API 주소를 입력 (끝 `/` 제외).
   예: `https://seller-margin-api-georgia-5lro.onrender.com`
3. **API URL 저장** 버튼 클릭.

---

## 4. 확인 순서 (요약)

| 순서 | 확인 내용 |
|------|-----------|
| 1 | Netlify에 **현재 로컬의** `seller-dashboard-v3.html` 배포 |
| 2 | 구글 시트 Apps Script를 **`apps-script/Code.gs` 전체**로 교체 후 웹 앱 재배포 |
| 3 | 대시보드 설정에서 **Apps Script URL**·**백엔드 API URL** 저장 |
| 4 | 브라우저 새로고침 후 **계산(시중가 조회)·소싱목록·판매관리·회계** 탭과 시트 구조 확인 |

이렇게 하면 **실제 웹페이지와 시트가 SYSTEM_DESIGN.md와 같게** 동작합니다.
