# 업데이트 후 실행 절차

> 코드/설계 업데이트를 반영한 뒤, 실제 서비스가 새 버전으로 동작하도록 하는 순서입니다.

---

## 실행 순서 요약

| 순서 | 대상 | 작업 | 확인 |
|------|------|------|------|
| 1 | 프론트엔드 | Netlify에 HTML 배포 | 로그인·계산 탭 동작 |
| 2 | 백엔드 | Render 배포 또는 환경변수 확인 | 시중가 조회 동작 |
| 3 | Apps Script | Code.gs 반영 후 웹 앱 재배포 | 구글 시트 저장·판매기록 |
| 4 | (선택) Google Cloud | OAuth 도메인/리디렉션 URI | 리디렉트 로그인 |
| 5 | 통합 확인 | 대시보드에서 한 번씩 실행 | 전 탭·저장·조회 |

---

## 1. 프론트엔드 (Netlify)

**수정한 파일:** `seller-dashboard-v3.html` 또는 `seller-dashboard-v4.html`

1. **Netlify** (https://app.netlify.com) 로그인 → 사용 중인 사이트 선택.
2. **Deploys** 탭에서:
   - **드래그 앤 드롭**: 로컬의 `seller-dashboard-v3.html`(또는 v4)을 드래그해 배포.
   - 또는 **Git 연동**인 경우: 해당 파일 수정 후 push → 자동 배포.
3. 배포 완료 후 **사이트 URL**로 접속.
4. **확인**: 로그인 화면 → 로그인 → 계산 탭에서 시중가 조회·마진 계산·저장 버튼 등 새 기능이 보이는지 확인.

**참고:** 상세 절차는 `docs/DEPLOY_SYNC.md` §1 참고.

---

## 2. 백엔드 (Render)

**수정한 파일:** `backend/main.py` 등

1. **Render** (https://dashboard.render.com) 로그인 → 해당 Web Service 선택.
2. **Manual Deploy** → **Deploy latest commit** (Git 연동 시),  
   또는 로컬에서 수정한 `main.py`를 반영한 뒤 push 후 자동 배포.
3. 배포 로그에서 **빌드·시작 성공** 확인.
4. **확인**:  
   - 브라우저에서 `https://[백엔드URL]/` 접속 → `{"status":"ok"}` 등 응답 확인.  
   - 대시보드 **설정** 탭에 백엔드 API URL이 올바르게 들어가 있는지 확인.  
   - **계산** 탭에서 상품명 입력 후 **시중가 조회** → 결과가 나오면 백엔드 연동 정상.

**환경변수:** 네이버 API(NAVER_CLIENT_ID, NAVER_CLIENT_SECRET) 등이 Render 환경변수에 설정되어 있는지 확인.

## 3. Apps Script (구글 시트)

**수정한 파일:** `apps-script/Code.gs`

1. **구글 시트** 열기 (시트 ID: `1D6IlJquibWJfUkmIrKSz-PF4JYSa10dJd_GQdwtSSSg`).
2. **확장 프로그램 → Apps Script** 이동.
3. 기존 `Code.gs` 내용 **전부 선택 후 삭제**.
4. 로컬의 **`apps-script/Code.gs`** 전체 복사 → Apps Script 편집기에 **붙여넣기**.
5. **저장** (Ctrl+S).
6. **배포**:
   - **배포 → 새 배포** (또는 **배포 관리 → 편집 → 버전: 새 버전**).
   - 유형: **웹 앱**.
   - 실행 사용자: **본인**, 액세스: **모든 사용자**.
   - **배포** 클릭 후 나오는 **웹 앱 URL** 복사.
7. **대시보드** (Netlify에 배포한 페이지) → **설정** 탭 → **Step 3 — Apps Script 배포 URL**에 위 URL 입력 → **URL 저장**.
8. **확인**:  
   - 계산 탭에서 마진 계산 후 **☁️ 구글 시트에 저장** → 상품목록 시트에 행 추가되는지 확인.  
   - 판매관리·회계 탭에서 판매기록·매입매출 저장이 되는지 확인.

**추가:** `generateMonthlyReport`, `calculateSimplifiedVAT` 등 새 액션을 쓸 경우, 프론트에서 해당 `action`으로 요청하는지 확인.

**참고:** 상세 절차는 `docs/DEPLOY_SYNC.md` §2 참고.

## 4. (선택) Google Cloud Console — 로그인 도메인

**변경한 경우:** OAuth 클라이언트, 배포 URL(프론트 주소) 변경 시.

1. **Google Cloud Console** (https://console.cloud.google.com) → **API 및 서비스 → 사용자 인증 정보**.
2. 사용 중인 **OAuth 2.0 클라이언트 ID** 선택.
3. **승인된 JavaScript 원본**에 프론트 URL 추가  
   예: `https://ohnayu-sketch.github.io/seller-margin-api/`
4. **승인된 리디렉션 URI**에 같은 URL 추가 (리디렉트 로그인 사용 시).
5. **저장**.
6. **확인**: 시크릿 창에서 대시보드 접속 → 로그아웃 후 다시 로그인(GSI 버튼 또는 “Google로 로그인” 버튼)이 되는지 확인.

## 5. 통합 확인 (체크리스트)

업데이트 후 한 번씩 아래를 실행해 보세요.

- [ ] **로그인** — GSI 또는 커스텀 버튼으로 로그인 가능.
- [ ] **계산 탭** — 시중가 조회, 마진 자동 계산, 역산/손익분기점, 구글 시트 저장.
- [ ] **소싱목록** — 목록 로드, 필터, 판매 시작.
- [ ] **판매관리** — 판매기록 추가, 목록 표시.
- [ ] **회계** — 거래 입력, 목록·필터, (연동했다면) 연간 부가세 계산.
- [ ] **설정** — Apps Script URL, 백엔드 URL, 허용 이메일 저장.

## 문제 발생 시 (롤백)

1. **업데이트 기록 확인**: `작업지시/UPDATE_LOG.md` 에서 최근 변경 **변경 파일** 확인.
2. **롤백**: 해당 파일만 이전 버전(백업 또는 Git 이전 커밋)으로 복원 후, 위 **1~3** 순서대로 다시 배포.
3. **기능 단위 복원**: `작업지시/CHECKLIST_v3_5.md` 를 보고 필요한 기능만 다시 적용할지 결정.
