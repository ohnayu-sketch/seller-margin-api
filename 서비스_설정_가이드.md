## 1. GitHub 설정

### 1-1. 저장소 구조 (현재 권장)

```
my-seller-dashboard/
├── seller-dashboard-v3.html    ← 메인 앱 (Netlify 배포용)
├── main.py                      ← FastAPI 백엔드 (Render 배포용)
├── requirements.txt             ← Python 패키지 목록
└── README.md
```

### 1-2. 파일 업로드 방법

**웹에서 직접 (가장 쉬움):**
1. https://github.com 접속 → 해당 저장소 클릭
2. 파일 클릭 → 연필 아이콘(Edit) 클릭
3. 내용 붙여넣기 → **Commit changes** 클릭

**새 파일 올리기:**
1. 저장소 페이지 → **Add file** → **Upload files**
2. 파일 드래그 → **Commit changes** 클릭

### 1-3. 저장소가 없는 경우 새로 만들기

1. https://github.com → 우상단 **+** → **New repository**
2. Repository name: `seller-dashboard` (원하는 이름)
3. **Private** 선택 (중요: 코드 공개 방지)
4. **Create repository** 클릭
5. 파일 업로드 (위 방법 참고)

### 1-4. GitHub 주의사항

> ⚠️ **절대 public 저장소에 올리면 안 되는 것:**
> - 구글 Client ID (현재는 HTML에 하드코딩되어 있으나 노출 위험 낮음)
> - 네이버 API 키 (main.py의 `NAVER_CLIENT_ID`, `NAVER_CLIENT_SECRET`)
> - 구글 시트 Apps Script URL

**환경변수 처리 (main.py):**
```python
# main.py에서 이렇게 되어 있어야 함
import os
NAVER_CLIENT_ID = os.environ.get("NAVER_CLIENT_ID", "")
NAVER_CLIENT_SECRET = os.environ.get("NAVER_CLIENT_SECRET", "")
```
실제 키 값은 Render 환경변수에 입력 (아래 4번 참고)

## 2. Netlify 설정

**역할:** HTML 파일 1개를 웹사이트로 호스팅

### 2-1. 최초 배포 (GitHub 연동 방식)

1. https://netlify.com 접속 → 구글/GitHub으로 로그인
2. **Add new site** → **Import an existing project**
3. **GitHub** 선택 → 저장소 선택
4. 설정:
   - Branch to deploy: `main`
   - Build command: (비워두기)
   - Publish directory: `/` 또는 `.`
5. **Deploy site** 클릭
6. 배포 완료 후 URL 확인: `https://랜덤이름.netlify.app`

### 2-2. 파일 직접 드래그 배포 (가장 빠름)

1. https://netlify.com → **Sites** 탭
2. 기존 사이트 클릭 → **Deploys** 탭
3. 페이지 하단 **"Drag and drop your site output folder here"** 영역에
4. `seller-dashboard-v3.html` 파일이 들어있는 폴더 드래그 앤 드롭
5. 자동 배포 완료 (30초~1분)

> 💡 파일 하나만 있으면 폴더 안에 넣어서 드래그해야 함

### 2-3. 사이트 이름 변경 (선택사항)

1. Netlify → 해당 사이트 → **Site configuration**
2. **Change site name** → 원하는 이름 입력
3. URL이 `https://새이름.netlify.app` 으로 변경됨
4. ⚠️ 변경 시 Google Cloud Console 리디렉션 URI도 함께 수정 필요

### 2-4. 현재 배포 확인

- 사이트 URL: https://ohnayu-sketch.github.io/seller-margin-api/
- Deploys 탭에서 배포 이력 확인 가능
- 실패 시 로그 확인 가능

## 3. Google Cloud Console 설정 ⭐ 가장 중요

**역할:** 구글 로그인 버튼이 작동하려면 반드시 필요

### 3-2. 기존 OAuth 클라이언트 확인

1. 왼쪽 메뉴 → **APIs & Services** → **Credentials**
2. **OAuth 2.0 Client IDs** 섹션에서 현재 클라이언트 확인
3. 클라이언트 ID: `985307778387-1v16a641sg34lsmsdbliamfcettauto6.apps.googleusercontent.com`
4. 해당 항목 클릭해서 편집 화면 진입

### 3-3. 도메인 등록 (로그인 안 될 때 핵심 해결책)

편집 화면에서 **실제로 앱을 여는 주소**에 맞게만 넣으면 됩니다.

#### GitHub Pages로 배포한 경우 (예: ohnayu-sketch.github.io/seller-margin-api)

**① 승인된 JavaScript 원본 (Authorized JavaScript origins)**  
→ **경로 없이** 프로토콜 + 호스트만 입력 (끝에 `/` 없음)

```
https://ohnayu-sketch.github.io
```

**② 승인된 리디렉션 URI (Authorized redirect URIs)**  
→ **앱이 열리는 전체 URL**을 그대로 입력. 코드는 `origin + pathname`을 쓰므로 아래와 **완전히 동일**해야 함.

```
https://ohnayu-sketch.github.io/seller-margin-api/seller-dashboard-v3.html
```

- ❌ `https://.../seller-dashboard-v3.html/` (끝에 슬래시 있으면 안 됨)
- ❌ `https://ohnayu-sketch.github.io` 만 (경로 없으면 로그인 복귀 시 404)
- ✅ 브라우저 주소창에 보이는 그대로 한 줄 추가

#### Netlify로 배포한 경우

**① 승인된 JavaScript 원본**
```
http://localhost
https://ohnayu-sketch.github.io/seller-margin-api/
```

**② 승인된 리디렉션 URI**
```
https://ohnayu-sketch.github.io/seller-margin-api/
https://ohnayu-sketch.github.io/seller-margin-api/
```

4. **저장** 클릭
5. ⏰ **5~10분 기다린 후** 앱에서 로그인 시도

> ⚠️ 저장 후 즉시 적용되지 않음. 최대 10분 소요.

### 3-3-2. 로그인 안 될 때 확인 체크리스트

| 확인 항목 | 설명 |
|-----------|------|
| **JavaScript 원본** | `https://본인도메인` 만 (예: `https://ohnayu-sketch.github.io`). 경로·끝 슬래시 없음. |
| **리디렉션 URI** | 앱 페이지 **전체 URL**과 **완전 일치** (대소문자, 슬래시 개수까지). 브라우저 주소창 복사해서 붙여넣기 권장. |
| **OAuth 클라이언트 유형** | **웹 애플리케이션** (데스크톱/기타 아님). |
| **저장 후 대기** | 변경 후 5~10분 지난 뒤 다시 로그인 시도. |
| **시크릿/다른 브라우저** | 캐시·이전 설정 제거 후 테스트. |
| **디버그 모드 사용 시** | 주소에 `?oauth_debug=1` 붙이면 복귀 시 # 뒤 내용을 8초간 표시. **이때는 아래 리디렉션 URI를 콘솔에 하나 더 추가:** `https://본인도메인/경로/seller-dashboard-v3.html?oauth_debug=1` |

### 3-4. 사이트 URL이 바뀌었을 때

배포 주소(Netlify 사이트 이름, GitHub Pages 저장소/경로)를 바꾸었다면 반드시 여기도 업데이트:

2. 새 URL 추가

### 3-5. 새 OAuth 클라이언트 만들기 (처음부터 시작하는 경우)

1. **Credentials** 페이지 → **+ Create Credentials** → **OAuth client ID**
2. Application type: **Web application**
3. Name: `셀러 마진 계산기` (구분용)
4. 위 3-3 내용 동일하게 입력

8. Netlify에 재배포

### 3-6. OAuth 동의 화면 설정 (처음 설정 시)

1. **APIs & Services** → **OAuth consent screen**
2. User Type: **External** 선택 → **Create**
3. App name: `셀러 마진 계산기`
4. User support email: 본인 이메일
5. 스크롤 다운 → **Save and Continue** 반복 (나머지는 기본값)
6. **Test users** 단계에서 접속 허용할 구글 이메일 추가:
   - 남편 이메일

> 💡 앱이 "테스트 모드"면 테스트 유저로 등록된 이메일만 로그인 가능

## 4. Render 설정 (백엔드 Python 서버)

**역할:** 네이버쇼핑 API 조회, 타겟층 분석 등 백엔드 처리

### 4-0. Render 처음 설정하기 (GitHub 저장소 연결)

아직 Render에 서비스를 만들지 않았다면 아래 순서대로 진행하세요.

1. **Render 가입·로그인**
   - https://render.com 접속
   - **Get Started for Free** → **Sign in with GitHub** 선택
   - GitHub 계정(ohnayu-sketch)으로 로그인 후 Render 접근 권한 허용

2. **새 Web Service 만들기**
   - Render 대시보드에서 **New +** → **Web Service** 클릭
   - **Connect a repository**에서 **GitHub** 선택
   - 저장소 목록에서 **ohnayu-sketch/seller-margin-api** 선택 (안 보이면 **Configure account**에서 저장소 접근 허용)
   - **Connect** 클릭

3. **서비스 설정 (화면에 맞게 입력)**
   - **Name**: 원하는 이름 (예: `seller-margin-api`) — 나중에 URL이 `https://이이름.onrender.com` 이 됩니다.
   - **Region**: Singapore 또는 가까운 지역
   - **Branch**: `main`
   - **Root Directory**: 비워두기 (저장소 루트에 `main.py`, `requirements.txt`가 있으면 됨)
   - **Runtime**: **Python 3**
   - **Build Command**: `pip install -r requirements.txt` (기본값 그대로)
   - **Start Command**: `uvicorn main:app --host 0.0.0.0 --port $PORT` (기본값 그대로)
   - 저장소에 `render.yaml`이 있으면 위 항목 중 일부는 자동으로 채워질 수 있습니다.

4. **환경변수 추가 (배포 전에 꼭 설정)**
   - **Advanced** 펼치기 → **Add Environment Variable**
   - 아래 세 개 추가:

   | Key | Value |
   |-----|--------|
   | `NAVER_CLIENT_ID` | 네이버 개발자센터에서 발급한 Client ID |
   | `NAVER_CLIENT_SECRET` | 네이버 개발자센터에서 발급한 Client Secret |
   | `PORT` | `10000` (또는 비워두면 Render가 자동 지정) |

5. **배포 시작**
   - **Create Web Service** 클릭
   - 빌드·배포 로그가 나오며 2~5분 정도 걸립니다.
   - 상단에 **Your service is live at** 다음에 나오는 URL을 복사합니다. (현재: `https://seller-margin-api-georgia-5lro.onrender.com`)

6. **대시보드에 URL 입력**
   - Netlify에 배포한 셀러 마진 앱 → **설정** 탭
   - **백엔드 API URL** (또는 Step 3-2) 입력란에 방금 복사한 Render URL 붙여넣기 (끝에 `/` 없이)
   - 저장 후 **계산** 탭에서 시중가 조회로 동작 확인

### 4-1. 현재 배포 확인

- 서버 URL: **https://seller-margin-api-georgia-5lro.onrender.com**
- https://render.com 로그인 → Dashboard에서 해당 Web Service 클릭 → 상단 URL 확인

## 8. 문제 해결 빠른 참조

| 증상 | 원인 | 해결 |
|------|------|------|
| 구글 로그인 버튼 안 나옴 | Google Cloud Console 도메인 미등록 | 3-3 참고, 10분 대기 |
| 시중가 조회 안 됨 | Render 슬립 or API키 문제 | Render 로그 확인, 환경변수 재확인 |
| 저장이 안 됨 | Apps Script URL 만료 | 6-2 참고, 새 배포 URL 입력 |
| 앱이 구버전으로 보임 | 브라우저 캐시 | 강력 새로고침 (주소창 재입력) |
| Render 첫 요청 느림 | 무료 플랜 슬립 | 정상, 두 번째 요청부터 빠름 |
| 로그인은 되는데 데이터 없음 | Script URL 미입력 | 설정 탭에서 URL 입력 |
