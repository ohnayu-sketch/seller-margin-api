# 업데이트 기록: 구글 OAuth 인증 전면 전환 및 관리자 초대 시스템 도입 (2026-03-02)

## 1. 개요 및 목적
* 기존의 단순 퍼블릭 비밀번호(`0000`) 방식은 보안에 취약하므로 이를 삭제하고, 오직 지정된 구글 계정으로만 접근할 수 있는 **강력한 화이트리스트(Whitelist)** 기반의 인증 체계로 전환함.
* 모바일 기기별로 인증을 부여하기 위해 **URL 파라미터 기반의 '초대 링크'** 시스템을 독자적으로 개발하여 반영함.

## 2. 세부 개발 및 적용 내역
1. **비밀번호 로직 및 UI 전면 폐기**
   * 로그인 화면의 비밀번호 입력창, 비밀번호 초기화 버튼, 설정 탭의 비밀번호 변경 UI를 모두 HTML에서 제거함.
   * `doSimpleLogin`, `getAppPassword`, `changeAppPassword` 등의 자바스크립트 함수들을 구동부에서 완전 삭제함.
2. **이메일 화이트리스트 접근 제어 (`isAllowed` 함수 업그레이드)**
   * 남편(`ohnayu@gmail.com`)과 아내(`ohhhjs90@gmail.com`)의 이메일을 하드코딩된 **영구 관리자(Admin)**로 지정.
   * `bypass@local` (테스트용 프리패스 우회) 로직을 폐기하여 허가받지 않은 모든 접근을 원천 차단.
3. **[Admin] 초대 링크 발급 기능 신설**
   * 설정 탭에 관리자 전용 **'접근 권한 초대 링크 발급'** 패널 추가.
   * 텍스트 박스에 직원의 구글 이메일을 입력하고 [링크 복사]를 누르면 `?invite=직원이메일` 형태의 고유 URL이 복사되는 `generateInviteLink()` 함수 구현.
   * 초대 링크를 통해 접속한 브라우저는 해당 이메일을 로컬 스토리지(`invited-email`)에 영구 저장하여, 이후 구글 로그인 시 자동으로 승인되도록 파싱 로직 구현.
4. **안전망: 자동 백업 폴더 신설**
   * 향후 시스템 수정 시 원본 유실을 방지하기 위해 `backup_dashboard/` 디렉토리를 신설.
   * 코드를 업데이트하기 직전에 항상 날짜와 시간이 기재된 사본을 자동으로 보관하도록 업무 프로세스를 정립함.
