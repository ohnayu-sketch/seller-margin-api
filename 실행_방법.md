# 셀러 마진 계산기 — 실행 방법

## 1. 처음 한 번만 할 설정 (이미 했다면 2번으로)

### 1-1. GitHub에 올리기
- 저장소: 예) `https://github.com/ohnayu-sketch/seller-margin-api`
- **필수 파일 (저장소 루트에 두기)**:
  - `seller-dashboard-v3.html` ← 실제 앱(메뉴·계산 전부 포함). **반드시 루트에** 있어야 함.
  - `index.html` ← 접속 시 위 파일로 넘겨 주는 용도.
- **경로**: 두 파일 모두 **프로젝트 최상단**(루트). `docs/`나 다른 폴더 안에만 있으면 안 됨.
- **GitHub Pages 켜기**: 저장소 → **Settings** → **Pages** → Source를 **main** 브랜치, Folder **/ (root)** 선택 후 저장.
- 자세한 체크리스트는 `GitHub_배포_필수파일.md` 참고.

### 1-2. Google Cloud Console
- **APIs & Services** → **Credentials** → OAuth 2.0 클라이언트 ID 선택
- **승인된 JavaScript 원본**에 한 줄 추가:
  ```
  https://ohnayu-sketch.github.io
  ```
- **승인된 리디렉션 URI**에 아래를 **그대로** 추가 (본인 주소에 맞게 도메인/경로만 수정):
  ```
  https://ohnayu-sketch.github.io/seller-margin-api/seller-dashboard-v3.html
  ```
- 저장 후 **5~10분** 대기

### 1-3. (선택) 시트·백엔드
- **구글 시트**: Apps Script에 `apps-script/Code.gs` 붙여넣고 배포 → 웹 앱 URL 복사
- **백엔드**: Render 등에 `backend/` 배포 후 API URL 확인  
→ 앱 **설정** 탭에서 위 URL들 입력

---

## 2. 실행 = 브라우저에서 열기

**실행**은 “프로그램을 켜는 것”이 아니라 **브라우저에서 주소를 여는 것**입니다.

1. **Chrome 등 브라우저**를 연다.
2. 주소창에 아래 주소를 **그대로** 입력하거나, 즐겨찾기한 주소를 연다.
   ```
   https://ohnayu-sketch.github.io/seller-margin-api/seller-dashboard-v3.html
   ```
3. **Enter** → 로그인 화면이 뜨면 **비밀번호** 입력 (기본 **0000**) → **로그인** 버튼 또는 Enter.
4. 로그인되면 대시보드(계산·소싱목록·판매 등)가 보이면 **실행 완료**.

---

## 3. 코드 수정 후 다시 실행하려면

1. **로컬에서** `seller-dashboard-v3.html` 등 수정
2. **GitHub에 push** (파일 업로드 또는 Git push)
3. **1~2분** 정도 기다린 뒤
4. 브라우저에서 **같은 주소** 다시 열기 (또는 **Ctrl+Shift+R**으로 강력 새로고침)

→ GitHub Pages가 새 파일로 갱신되면, 그때부터 “다시 실행”한 것과 같습니다.

---

## 4. 앱에 안 들어가는 문제 (실행 문제) — 체크 순서

주소를 열었는데 로그인 화면에서 멈추거나, 구글 계정 선택 후 다시 로그인 화면으로 돌아오거나, **400 redirect_uri_mismatch** 가 나올 때 아래 순서대로 확인하세요.

### 4-1. 400 redirect_uri_mismatch 가 나올 때
- **원인**: 앱이 구글에 보내는 주소가 Google Cloud Console에 없음.
- **해결**:
  1. 로그인 화면 맨 아래에 나오는 **“아래를 그대로 추가”** 주소를 **복사**.
  2. Google Cloud Console → **Credentials** → OAuth 2.0 클라이언트 ID → **승인된 리디렉션 URI**에 **+ URI 추가**로 그대로 붙여넣기 (끝에 `/` 없이).
  3. **저장** 후 **5~10분** 기다렸다가 다시 로그인.

- **디버그 주소**(`?oauth_debug=1` 붙여서 연 경우)로 로그인했다면, 그 **전체 주소**도 리디렉션 URI에 따로 추가해야 함.  
  예: `https://ohnayu-sketch.github.io/seller-margin-api/seller-dashboard-v3.html?oauth_debug=1`

### 4-2. 계정 선택 후 다시 로그인 화면으로 돌아올 때
- **확인 1**: **승인된 리디렉션 URI**가 **한 글자도 틀리지 않게** 등록돼 있는지 (끝 `/` 없음, `http`/`https` 구분).
- **확인 2**: 수정한 `seller-dashboard-v3.html`을 **GitHub에 push**했는지. (JWT 파싱 수정 등이 반영돼 있어야 함.)
- **확인 3**: 콘솔에서 URI 저장한 뒤 **5~10분** 지났는지.
- **확인 4**: **시크릿 창**에서 열거나 **Ctrl+Shift+R**(강력 새로고침) 후 다시 로그인.

### 4-3. “구글 로그인 버튼 불러오는 중…” 에서만 멈출 때
- **공식 버튼**(GSI)이 안 뜨는 환경이면 **아래 흰색 “Google로 로그인” 버튼**을 누르면 됨.
- 약 **3초** 지나면 문구가 “아래 Google로 로그인 버튼을 눌러주세요.” 로 바뀜. 그때 아래 버튼 클릭.

### 4-4. 구글 인증 없이 들어가기 (우회)
구글 로그인/리디렉션 문제로 앱에 들어가지 못할 때, **임시로** 인증 없이 진입하는 방법입니다. **PC·휴대폰 모두 동일**하게 사용 가능합니다.

- **휴대폰에서**: 주소 끝에 **`?skip_auth=1`** 붙인 URL을 **그대로** 주소창에 입력하거나, 그 주소를 **북마크/홈 화면에 추가**해 두고 열기.  
  예: `https://ohnayu-sketch.github.io/seller-margin-api/seller-dashboard-v3.html?skip_auth=1`
- **PC에서**: 위와 같이 주소에 `?skip_auth=1` 붙여서 열기.
- **(선택)** 주소에 **`?dev=1`** 붙인 뒤 로그인 화면에서 **「🔓 로그인 없이 들어가기」** 링크 클릭해도 됨.

한 번 들어가면 같은 탭(브라우저)에서는 세션 유지. 로그아웃하면 다시 로그인 화면으로 돌아갑니다.

### 4-5. 한 번에 정리
| 증상 | 할 일 |
|------|--------|
| **400 redirect_uri_mismatch** | 로그인 화면에 나온 주소를 콘솔 **승인된 리디렉션 URI**에 그대로 추가 → 저장 → 5~10분 후 재시도 |
| **계정 선택 후 로그인 화면으로 복귀** | 리디렉션 URI 정확히 등록, 최신 HTML push 여부, 5~10분 대기, 시크릿/강력 새로고침 |
| **버튼만 안 뜨고 진행 안 됨** | 아래 “Google로 로그인” 버튼 사용, 3초 후 안내 문구 확인 |
| **일단 앱부터 쓰고 싶을 때** | 주소에 `?skip_auth=1` 붙여서 열기 (우회 진입) |

### 4-5-2. 링크 들어갔는데 구글 로그인 창이 계속 나올 때
- **원인**: GitHub Pages에 **예전 버전**의 `seller-dashboard-v3.html`이 올라가 있음. (비밀번호 로그인으로 바꾼 파일이 아직 배포되지 않음.)
- **해결**:
  1. **지금 수정한** `seller-dashboard-v3.html` (비밀번호만 입력하는 버전)을 **GitHub 저장소에 push**.
  2. 1~2분 기다린 뒤, 브라우저에서 **강력 새로고침** (PC: **Ctrl+Shift+R**) 또는 **시크릿 창**으로 같은 링크 다시 열기.
- **구글 콘솔**: 비밀번호 로그인만 쓰면 **구글 설정은 수정·삭제하지 않아도 됨.** 나중에 구글 로그인을 다시 쓸 때를 위해 두어도 됨. 정리하고 싶다면 아래 "구글 콘솔 설정 정리" 참고.

### 4-6. 비밀번호 입력해도 웹/앱에 안 들어갈 때 (현재 로그인 방식)

아래 순서대로 확인하세요.

1. **주소 확인**  
   `https://ohnayu-sketch.github.io/seller-margin-api/seller-dashboard-v3.html` 인지 확인. (끝에 `/` 없음.)

2. **비밀번호**  
   - 처음 쓰는 경우 **0000** (숫자 네 자리) 정확히 입력.  
   - 이전에 설정에서 바꿨다면 **변경한 비밀번호** 입력.

3. **최신 페이지인지**  
   - 수정한 `seller-dashboard-v3.html`을 **GitHub에 push**했는지 확인.  
   - **강력 새로고침**: PC는 **Ctrl+Shift+R**, 휴대폰은 브라우저에서 **새로고침** 또는 **시크릿/비공개** 창으로 같은 주소 열기.

4. **비밀번호를 잊었을 때**  
   - 주소 끝에 **`?reset_pw=1`** 붙여서 열기.  
     예: `https://ohnayu-sketch.github.io/seller-margin-api/seller-dashboard-v3.html?reset_pw=1`  
   - 로그인 화면에 **「비밀번호를 0000으로 초기화」**가 보이면 클릭.  
   - 이후 **0000**으로 다시 로그인.

5. **그래도 안 되면**  
   - 같은 주소를 **시크릿(비공개) 창**에서 열어서 **0000**으로 로그인 시도.  
   - 다른 브라우저(Chrome, Safari 등)로 시도.

### 4-7. 구글 콘솔 설정 정리 (선택, 비밀번호 로그인만 쓸 때)
비밀번호 로그인만 사용한다면 **구글 OAuth 설정을 꼭 지울 필요는 없습니다.** 동작에는 영향 없음. 정리하고 싶을 때만 아래처럼 하면 됩니다.

- **리디렉션 URI / JavaScript 원본만 비우고 싶을 때**  
  1. https://console.cloud.google.com → **APIs & Services** → **Credentials**  
  2. 사용 중인 **OAuth 2.0 클라이언트 ID** (예: 셀러마진) 클릭  
  3. **승인된 JavaScript 원본**, **승인된 리디렉션 URI** 항목에서 기존 URI를 하나씩 삭제 (휴지통 아이콘)  
  4. **저장** 클릭  

- **OAuth 클라이언트 자체를 삭제하고 싶을 때**  
  1. **Credentials** 페이지에서 해당 **OAuth 2.0 Client ID** 오른쪽 **⋮** (또는 휴지통) 클릭  
  2. **Delete** 선택 후 확인  

→ 삭제해도 비밀번호 로그인에는 영향 없습니다.

## 5. 한 줄 요약

| 하고 싶은 것           | 하는 일 |
|------------------------|----------|
| **앱 실행**            | 브라우저에서 `https://ohnayu-sketch.github.io/seller-margin-api/seller-dashboard-v3.html` 열기 |
| **수정 반영 후 실행**  | 수정한 파일을 GitHub에 push → 1~2분 후 같은 주소로 다시 열기 |
| **앱에 안 들어갈 때** | **비밀번호 로그인**이면 **4-6** 확인 (0000, 강력 새로고침, `?reset_pw=1`로 초기화). 구글 로그인 문제면 **4-1~4-4** 참고. |

**서버를 따로 켜거나, 터미널에서 명령을 실행할 필요는 없습니다.**  
항상 “브라우저에서 위 주소를 연다”가 실행 방법입니다.
